Actividad 6

En esta actividad se investigo que es y como se elimina un controlador de dominio y objetos que administra un dominio.

¿Qué es un controlador de dominio?

Los administradores de red tienen presente que para aprovechar el potencial de los equipos con Windows Server es por medio de la creación de dominios.

Los dominios almacenan de manera centralizada la información de los usuarios como sus cuentas, los dispositivos como impresoras, directorios, etc. De igual manera facilitan las tareas del adminsitrador proporcionando seguridad.
En Windows Server el controlador de dominio es conocido como Active Directory (AD).

Caracteristicas de un controlador de dominio

Los controladores de dominio eliminan las conjeturas y las molestias de administrar computadoras y dispositivos en su red al “conectarlos” a un sistema maestro. Esto le permite a su administrador establecer salvaguardas, permisos y otros protocolos de acceso de forma remota en cada dispositivo.

  • Dar acceso solo a aquellos que lo necesitan

    El controlador de dominio le permite crear configuraciones mediante las cuales los usuarios pueden iniciar sesión en cualquier computadora que esté en su dominio y sus privilegios de seguridad los seguirán y se configurarán automáticamente en cualquier computadora.

  • Evite las infracciones de datos de “error del operador”.

    Un controlador de datos le permite evitar contraseñas demasiado simples al crear reglas para toda la red que requieren que los usuarios creen contraseñas únicas y complejas.

  • La gestión centralizada reduce los costos.

    Tener la capacidad de establecer parámetros de inicio de sesión y seguridad para todos los dispositivos desde un concentrador central reduce el tiempo y el costo de tener que proteger y configurar cada dispositivo individualmente, lo que puede llevar horas en comparación con solo minutos desde un controlador de dominio.

  • Recursos informáticos compartidos.

  • Cerrar el acceso no autorizado

    - Bloquee las cuentas de usuario con demasiados intentos fallidos de inicio de sesión.
    - Inhabilite las cuentas de usuario inmediatamente cuando un empleado abandone su empresa.
    - Configure automáticamente todas las computadoras para bloquear la pantalla después de un período de inactividad establecido.
    - Requerir contraseñas de inicio de sesión para pantallas bloqueadas.
    -Restrinja el acceso USB (unidad flash) por permisos de usuario.


Eliminación de un controlador de dominio

Para eliminar el controlador de un dominio se ejemplificará con Windows server:

  • Iniciamos sesión como Administrador del equipo y desde Inicio > Ejecutar ejecutamos el comando dcpromo. Se nos muestra una pantalla de inicio del asistente de confi-guración de DA.
  • Si nuestro dominio contase con algún otro bosque, con controladores adicionales de dominio o con subdominios, no podríamos elimi-narlo. En este caso tendríamos que ir eliminando los controladores de dominio jerárquicamente de menor a mayor jerarquía.Teniendo esto claro, pulsamos Aceptar
  • Pulsamos Siguiente y aparece una pantalla dándonos información sobre las particiones en las que se ha replicado la base de datos de DA. Pul-samos Siguiente y sale una nueva pantalla en la que indicaremos si queremos eliminar o no toda la informa-ción que las particiones de la base de datos contienen.
  • Una vez marcada la casilla Eliminar todas las particio-nes, pulsamos en Siguiente.
  • Marcamos la casilla de verificación que indica esta circuns-tancia y pulsamos Siguiente.
  • Pulsamos Siguiente en las pantallas posteriores y el pro-ceso de despromoción comienza. Pasados unos minu-tos, cuando finalice, tendremos que reiniciar el equipo o marcar la casilla Reiniciar al completar de la última pantalla que aparece.
    • Objetos que administra un dominio
  • Usuarios globales

    - Identificar y autentificar a las personas (usuarios) que podrán acceder al sistema, es decir, loguearse al controlador de dominio. -Administrar y gestionar los permisos y privilegios que servirán para controlar el acce-so que los usuarios hacen a los recursos del sistema.

  • Grupos

    En el controlador de dominio pueden crearse diferentes tipos de grupos de usuarios: grupos de distribución y grupos de seguridad

  • Equipos

    La misma base de datos del Directorio Activo de un dominio que conserva toda la infor-mación relativa a cuentas de usuarios y grupos globales almacena asimismo informa-ción relativa a los equipos (computers) que forman parte del dominio.

  • Unidades organizativas

    Son objetos del directorio que a su vez pueden contener otros ob-jetos. El uso fundamental de las UO es delegar la administración de sus objetos a otros usuarios distintos del administrador del dominio y personalizar el comportamiento de los usuarios y/o equipos mediante la aplicación de directivas
